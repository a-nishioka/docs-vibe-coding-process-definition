> 複数サイクルにわたる機能ロードマップを管理するドキュメント。
> 各タスクは「タスクID」で識別し、詳細計画（tasks/T-XXX_<機能名>.md）と紐付ける。
>
> Claudeは以下のタイミングで必ずこのファイルを参照・更新すること：
> - 【参照】フェーズ1 要件定義の開始時

> - 【更新】フェーズ5 フィードバックの承認後（ロードマップへの影響があれば）

## Claudeへの指示

### 要件定義フェーズ開始時
1. 「ロードマップ > 現在のサイクル」を確認し、対象タスクIDをユーザーに確認する
2. 対象タスクの「タスク定義」セクションを読み、要件定義のベースにする
3. `tasks/T-XXX_<機能名>.md` を新規作成し、要件定義の内容を記入する
4. 要件定義サマリーを提示し承認を得る
5. スコープ・順序の変更があれば「更新ログ」に記録してロードマップを修正する

### フィードバックフェーズ承認後
1. フィードバックがロードマップに影響するか検討する
2. 影響する場合は変更案を提示し承認を得てから更新ログに記録する：
   ```
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   📋 全体計画への影響確認
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   フィードバック : [要約]
   変更対象      : T-XXX [タスク名]
   変更内容      : [変更前] → [変更後]
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   ▶ 全体計画を更新しますか？
   ```
3. 影響なしの場合も「影響なし」と明示的に報告する
