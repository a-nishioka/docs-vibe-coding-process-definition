# DECISIONS.md — 技術的意思決定ログ（ADR）

> 重要な技術的決定を記録する。
> Claudeは新しい設計判断を行った際、このファイルへの記載をユーザーに提案すること。

---

## ADRの書き方

```
## ADR-XXX: [決定のタイトル]
- **日付**: YYYY-MM-DD
- **ステータス**: 提案中 / 採用 / 却下 / 廃止
- **背景**: なぜこの決定が必要だったか
- **決定内容**: 何を決定したか
- **理由**: なぜそれを選んだか
- **トレードオフ**: 捨てたものは何か
- **影響範囲**: どこに影響するか
```

---

## 記録

### ADR-001: [タイトルの例]
- **日付**: YYYY-MM-DD
- **ステータス**: 採用
- **背景**: [背景を記入]
- **決定内容**: [決定を記入]
- **理由**: [理由を記入]
- **トレードオフ**: [捨てたものを記入]
- **影響範囲**: [影響範囲を記入]
